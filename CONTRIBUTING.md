# Contributing

Create a focused branch, use Conventional Commits, add tests for behavior changes, and run the
locked quality suite before opening a pull request. Contributions use Developer Certificate of
Origin 1.1: include `Signed-off-by` in every commit to certify your right to submit the work.

Architecture, contract, persistence, authentication, worker, or deployment changes require an
architecture proposal and compatibility notes. Follow the Code of Conduct and security policy.
